## Рефакторинг кода

Для проведения автоматизированного тестирования был выполнен рефакторинг двух модулей:

**Модуль авторизации (LoginPage.xaml.cs)** — логика авторизации вынесена из обработчика `BtnLogin_Click` в отдельный публичный метод `Auth(string login, string password)`, возвращающий `bool`. Метод проверяет корректность входных данных и наличие пользователя в базе данных.

**Модуль регистрации (RegisterPage.xaml.cs)** — логика регистрации вынесена из обработчика `BtnRegister_Click` в отдельный публичный метод `Register(string fullName, string login, string password)`, возвращающий `bool`. Метод проверяет заполненность полей, уникальность логина и добавляет нового пользователя в БД.

---

## Тестовые методы

### Тесты авторизации (AuthTests.cs)

| Метод | Тип | Описание |
|-------|-----|----------|
| `AuthTest()` | Смешанный | Проверка корректных данных (admin/123), несуществующего пользователя (user1/12345), пустых строк и пробелов |
| `AuthTestSuccess()` | Позитивный | Авторизация всех пользователей из БД: admin, user1, 666, 123 |
| `AuthTestFail()` | Негативный | Пустые поля, пробелы, несуществующие пользователи, неверный пароль при верном логине, неверный логин при верном пароле, спецсимволы, SQL-инъекция, длинные строки |

### Тесты регистрации (RegisterTests.cs)

| Метод | Тип | Описание |
|-------|-----|----------|
| `RegisterTestSuccess()` | Позитивный | Регистрация новых пользователей с уникальными логинами |
| `RegisterTestFail()` | Негативный | Пустые поля, пробелы, попытка регистрации с занятым логином (admin) |

## Выводы

В ходе практической работы были созданы автоматизированные unit-тесты для модулей авторизации и регистрации WPF-приложения «CinemaApp» с использованием MSTest в Microsoft Visual Studio.

Для обеспечения возможности тестирования был проведён рефакторинг кода: логика авторизации и регистрации была вынесена из обработчиков событий кнопок в отдельные публичные методы `Auth()` и `Register()`, возвращающие значение типа `bool`.

**Тесты авторизации:**
- `AuthTest` — пройден успешно. Смешанный тест, проверяющий как корректную авторизацию (admin/123), так и негативные случаи (несуществующий пользователь, пустые строки, пробелы).
- `AuthTestSuccess` — пройден успешно. Все 4 пользователя из базы данных (admin, user1, 666, 123) успешно авторизованы, так как их логины и пароли совпадают с записями в таблице Users.
- `AuthTestFail` — пройден успешно. Метод `Auth()` корректно возвращает `false` при пустых полях, пробелах, несуществующих пользователях, неверных комбинациях логин/пароль, спецсимволах и попытках SQL-инъекций.

**Тесты регистрации:**
- `RegisterTestSuccess` — пройден успешно. Новые пользователи с уникальными логинами корректно добавляются в базу данных.
- `RegisterTestFail` — пройден успешно. Метод `Register()` корректно отклоняет регистрацию при пустых полях, пробелах и попытке зарегистрироваться с уже занятым логином (admin).

Тестирование методом «белого ящика» подтвердило корректность обработки граничных случаев в модулях авторизации и регистрации приложения.
